Project Journey Recap

A brief look at the trajectory of the internship project from inception to delivery.

1. Problem Identified

Retail customers face high friction inquiring about product sizes and stocks at counter displays, leading to long cashier queues and store staff fatigue.

2. In-Store Research

Conducted counter observations: 42% of customer queries focused on size availability. Staff spent an average of 3 minutes checking rear stock rooms per inquiry.

3. System Designed

Designed a lightweight, mobile-first web portal requiring zero installs. Customers scan counter QR codes to instantly see live, real-time stock lists.

Key Vision: Enable friction-free retail shopping by placing real-time inventory at the customer's fingertips.

Final System Overview

A high-level summary of features planned, delivered, and scoping outcomes.

Outcome Statement

Delivered a fully functional, mobile-responsive Live Store Product Catalog featuring instant search autocompletion, category breakdowns, a secure Admin Stock Portal with an indicator mode, and custom counter QR code generators.

Scope Adjustment (Descope)

Rationale: The store operates as a physical counter retail shop. Customers inspect products live; adding a digital checkout would conflict with physical sales flow and POS registers.

Code Highlight: Real-time Autocomplete Dropdown

Custom search query handler highlighting technical implementation without external dependencies.

// Search input event listener with debounce
function initSearchSuggestions() {
  const searchInput = document.getElementById('searchBar');
  let debounceTimeout;

  searchInput.addEventListener('input', (e) => {
    clearTimeout(debounceTimeout);
    const query = e.target.value.trim().toLowerCase();

    debounceTimeout = setTimeout(() => {
      if (query.length < 2) {
        hideDropdown();
        return;
      }
      const filtered = products.filter(p => 
        p.name.toLowerCase().includes(query) ||
        p.category.toLowerCase().includes(query)
      );
      renderSuggestions(filtered);
    }, 200); // 200ms debounce buffer
  });
}
Why this is interesting

Instead of querying the database on every character stroke (which causes heavy network loads and UI stuttering on mobile), we implemented a lightweight local cache check with a 200ms debounce.

This guarantees the catalog page renders suggestions in under 15ms after typing pauses, assuring a premium customer experience even on spotty store cellular networks.

Technical Challenges & Solutions (Part 1)

Overcoming real-time search performance and rendering constraints.

Challenge: Search Stuttering

The Problem: On mobile screens, rapid keystrokes led to laggy DOM updates in the catalog list, freezing the keyboard.

First Attempt: Fetching and redrawing all products on each keystroke (failed due to high DOM write operations).

Resolution: Implemented a DocumentFragment container to render suggestion dropdown lists offline, swapping in the complete DOM node in a single write operation.

The Learning Outcome

This challenge highlighted that DOM updates are the primary bottleneck in mobile web rendering, not JavaScript computation speed.

Applying DocumentFragments and decoupling UI paint operations from search queries resolved keyboard freeze entirely, dropping search rendering speed from 240ms to 8ms.

Technical Challenges & Solutions (Part 2)

Implementing counter-level QR code generation and printing.

Challenge: Blurry QR Codes

The Problem: When managers printed out the generated QR codes, barcode scanners on phones failed to read them due to canvas image blur when scaling.

First Attempt: Exporting HTML5 canvas nodes to PNG/JPEG (blurry on high-res store laser printers).

Resolution: Swapped to generating the QR code layout entirely as an inline SVG tag with mathematical vectors, accompanied by print CSS variables.

The Print Resolution

SVG files scale infinitely without losing quality. We integrated a dedicated `@media print` rule styling the printable screen.

This hides dashboard sidebars and headers, exporting a perfectly sharp, centered vector QR card with matching store counter branding ready to cut and display.

Learnings & Personal Growth

Key takeaways from both a software engineering and professional perspective.

Technical Engineering Learnings
  • Vanilla JS Strength: Gained deep appreciation for DOM operations, understanding that massive web frameworks are not always required for high-speed mobile pages.
  • Mobile-First Optimization: Designing around cellular data speeds forced lightweight caching and minimal asset sizing (SVG vs PNG).
  • Secure Offline States: Learning how to utilize local persistence storage safely while preventing client-side data leaks.
Professional & Retail Insights
  • UX in Retail: Customer attention span in a busy physical store is extremely brief. Clean simplicity always wins over complex multi-step features.
  • Iterative Alignment: Regular client syncs helped identify that online checkout was unnecessary, preventing wasted weeks of backend development.
  • Diligence & Closure: Recognizing that code is only finished when testing, architecture layouts, and code reviews are completely documented.

System Limitations & Future Scope

Honest reflections on what the current system lacks and plans for future updates.

Current System Limitations
  • Manual Inventory Sync: Stock levels must be updated manually by managers inside the Admin Console, which is prone to human typing errors.
  • Lack of Multi-Store Support: The system is configured for a single storefront instance and does not separate warehouse reserves from display shelf stock.
  • Local Cache Persistence: Admin sessions depend on local browser state caches rather than persistent server tokens.
Future Scope & Roadmap
  • POS Barcode Integration: Sync database directly with the physical store POS checkout system, automatically reducing stock counts on checkout.
  • Multi-Manager Role Control: Add role-based authentication layers separating head manager (who can delete stock) from shop assistants (who only adjust stock counts).
  • Dynamic Analytics Dashboard: Provide analytics charts for store owners showing which counter QR codes get scanned most often during peak hours.
